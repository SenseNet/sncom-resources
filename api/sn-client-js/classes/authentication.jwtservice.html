    It is possible to send authentication requests using this action. You provide the username and password and will get the User object as the response if the login operation was successful or HTTP 403 Forbidden message if it wasnâ€™t. If the username does not contain a domain prefix, the configured default domain will be used. After you logged in the user successfully, you will receive a standard ASP.NET auth cookie which will make sure that your subsequent requests will be authorized correctly.

    The username and password is sent in clear text, always send these kinds of requests through HTTPS.

    Parameters

 complete: () => console.log('done'),
